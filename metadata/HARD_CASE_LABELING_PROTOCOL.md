# Hard Test 场景初标说明

这不是重新标注检测数据。原图片、YOLO 框、类别和数据划分均未改变；`hard_case_index.csv` 只为每张测试图增加一个场景级 `case_type`，用于分组诊断。

## 初标过程

1. 使用 `tools/build_hard_case_annotation_sheets.py` 生成只含原图和文件名的本地联系表，标注阶段不并排显示模型预测。
2. 逐页检查 146 张 Hard Test 原图，发现它们主要来自 7 个连续帧场景，存在大量相邻近重复。
3. 按连续场景的主要视觉困难统一初标；不根据 Baseline/ECA/LECA 的成败修改标签。
4. 标注者此前参与过少量模型案例审计，因此这不是严格盲法。当前也不是双人标注，没有一致性统计。答辩前建议由未接触预测、熟悉现场的人员复核，尤其是 `reflection` 与 `mixed`。

## 当前场景组

| 场景组 | 帧号范围 | 初标类型 | 判定依据 |
| --- | --- | --- | --- |
| S01 | 1–13 | structural_clutter | 广角底盘结构，孔洞和支架密集 |
| S02 | 21–43 | reflection | 金属区域有明显局部高亮，同时存在孔洞 |
| S03 | 44–69 | mixed | 孔洞、支架和大型圆形构件共同出现 |
| S04 | 70–100 | hole | 连续竖幅帧，孔洞为主要相似结构 |
| S05 | 114–194 | hole | 多处圆孔和黑色圆形结构 |
| S06 | 195–220 | normal | 螺栓近景较清楚，干扰相对较少 |
| S07 | 222–232 | circular_plate | 螺栓邻近大型圆形金属构件与孔洞 |

Hard Test 中没有形成可靠的 `low_light` 或 `rivet` 子集，因此不能用当前数据回答暗光或铆钉场景的真实分组性能。受控变暗实验只能称为合成光度压力测试，不能冒充真实低照度数据。
